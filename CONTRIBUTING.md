# Contributing to noto

noto へのコントリビューションを歓迎します。

## 貢献方法

1. **Issue を作成** — バグ報告や機能リクエストを GitHub Issues で共有
2. **フォーク** — リポジトリをフォーク
3. **ブランチを作成** — `feature/your-feature-name` などのブランチで開発
4. **コミット** — わかりやすいコミットメッセージで commit
5. **Pull Request を作成** — 説明を添えて PR を submit

## コーディング規約

- React Hooks を優先（クラスコンポーネントは避ける）
- 一つのコンポーネント = 一つのファイル
- CSS は BEM風の命名規則
- コメントは日本語で

## テストについて

現在、テストフレームワークは未導入です。PR の際は手動での動作確認をお願いします。
